# 更新记录

- 文档补充路径关键词匹配口径：跳过路径关键词和只扫描路径关键词按完整路径字符串检测。

- 新增 `发布说明.md`，整理适合朋友圈或转发给普通用户的简介、启动方式和使用步骤。

## 2026-06-12

- 设置文件固定到项目根目录 `settings.json`，并在仓库中保留默认配置；打包后使用 exe 同级目录 `settings.json`，不再读取 `%USERPROFILE%\.fileaudit\settings.json`。
- 新增清空扫描结果功能：扫描配置页的“清空结果”现在会清空概览、文件明细、重复文件、可疑文件和扫描错误，并重置顶部、侧边栏和底部状态。
- 新增扫描错误页，独立展示扫描过程中捕获到的路径和错误原因。
- 新增隐私/跳过规则：
  - 隐藏文件可选择标记风险、跳过文件或忽略。
  - 大文件可选择标记风险、跳过文件或关闭，标记和跳过共用“大文件阈值”。
  - 跳过指定目录名。
  - 跳过指定文件名。
  - 跳过指定扩展名。
  - 跳过路径关键词。
- 新增“只扫描匹配规则”：
  - 按扩展名匹配。
  - 按文件名关键词匹配。
  - 按路径关键词匹配。
  - 按文件类型匹配。
  - 文件级跳过规则固定优先于只扫描规则。
- `ScanSummary` 新增 `skipped_files`、`skipped_dirs` 和 `skip_reasons`，概览和报告可以显示跳过统计。
- 调整目录统计口径：`total_dirs` 现在按实际进入扫描流程的目录计数。
- 调整风险分级：
  - 双扩展名伪装为高风险。
  - 脚本文件、隐藏文件、时间异常、路径过长、无扩展名大文件为中风险。
  - 普通可疑扩展名、临时/备份文件、空文件、大文件为低风险。
- 双扩展名判断更谨慎：只有前一个扩展名像文档、图片、压缩包等诱饵扩展名时，才标记为双扩展名伪装，避免 `CNCMaps.Renderer.exe`、`libglib-2.0-0.dll` 这类名称误报。
- 新增临时/备份文件识别：`.tmp`、`.temp`、`.cache`、`.bak`、`.old` 和 `~` 结尾文件会标记为低风险。
- 新增无扩展名大文件规则。
- 扫描概览新增更多图表：扩展名 Top 10、文件大小分布、风险原因 Top 10、最大文件 Top 10、重复文件组、修改时间分布和跳过原因。
- 新增 `DonutChart` 图表组件。
- 概览图表区域改为标签页，避免图表过多时页面拥挤。
- 设置页和扫描配置页改为可滚动布局，容纳更多规则配置。
- 设置页新增“报告导出完整路径”开关；关闭后 CSV 和 HTML 报告只写文件名或目录名，降低完整路径泄露风险。
- 报告导出新增跳过统计、更多图表和扫描错误预览。
- 导出页移除暂未生效的复选框，改为明确说明当前会生成完整报告包。
- 设置页和扫描配置页的数字输入新增 `QIntValidator` 和统一正整数解析，错误提示能明确指出哪个字段不合法。
- 新增 `tests/test_scanner.py`，覆盖风险检测、双扩展名误报、白名单扩展名、重复文件、跳过规则、只扫描规则、导出路径脱敏和扫描错误导出。
- 更新所有 Markdown 文档，记录当前功能、规则概念、架构变化、修改入口、临时问题状态和官方文档查阅点。
- 新增 `fileaudit/utils/`，集中管理文件类型分类、大小/时间/风险文案格式化、跳过原因文案和图表分布统计。
- UI、HTML/CSV 报告和扫描核心改为共用 `fileaudit/utils/`，减少重复逻辑。
- 新增 `tests/test_utils.py`，覆盖公共工具函数、分布统计和目录路径脱敏统计。
- 默认跳过 Windows 常见受保护目录：`System Volume Information`、`$RECYCLE.BIN`、`Recovery`、`Config.Msi`。
- 扫描遇到 `PermissionError` / `WinError 5` 时，错误信息改为更友好的“无权限访问”说明。
- 扫描测试增至 17 个，覆盖默认保护目录跳过和权限错误文案。
- 扫描概览和 HTML 报告新增三类图表：风险目录 Top 10、重复扩展名 Top 10、扫描错误目录 Top 10。
- 公共统计测试增至 18 个，覆盖新增图表分布函数。
- 设置页新增“修改时间分类”月份阈值，默认 3 个月；扫描概览和 HTML 报告的修改时间分布会使用该设置。
- 测试增至 20 个，覆盖默认月份设置、修改时间分布和报告输出。
- 设置页新增“单文件超时”阈值，默认 15 秒，`0` 表示关闭。
- 扫描核心新增慢文件/慢 Hash 跳过记录：单文件元数据读取或 Hash 计算超过阈值时记录到扫描错误和跳过原因，并继续扫描后续文件。
- 测试增至 22 个，覆盖超时错误文案、默认超时设置和慢文件/慢 Hash 跳过原因显示。
- 设置页和扫描配置页收敛重复入口：
  - 隐藏文件统一为“隐藏文件处理：标记风险 / 跳过文件 / 忽略”。
  - 大文件统一为“大文件处理：标记风险 / 跳过文件 / 关闭”，共用“大文件阈值”。
  - 移除规则优先级下拉框，跳过规则固定优先于只扫描规则。
- 新增设置冲突校验，覆盖可疑扩展名与白名单/跳过扩展名重叠、只扫描规则为空、只扫描路径落入已跳过路径等情况。
- 测试增至 30 个，覆盖设置冲突校验。
- 扫描配置页和设置页新增鼠标悬停解释，覆盖检测开关、处理模式、阈值输入、跳过规则、只扫描规则和导出完整路径开关。
- 扫描配置页和设置页新增实时“规则状态”提示，用户修改规则时会提前显示配置冲突，开始扫描和保存设置时仍保留最终拦截。
- 扫描配置页新增“预检查”按钮，后台估算预计扫描文件数、目录数、总大小、跳过数量、跳过原因和 Hash 候选文件数。
- 新增 `ScanPreview`、`preview_scan()` 和 `PreviewWorker`，预检查复用正式扫描的跳过/只扫描判断，但不生成完整记录、不计算 Hash。
- 扫描进度新增详细反馈：正式扫描时显示已扫描文件数、已访问目录数、已跳过数量、错误数和当前路径；Hash 阶段继续显示 `当前 / 总数`。
- 测试增至 32 个，覆盖预检查统计和详细扫描进度回调。
- 文件明细、重复文件、可疑文件和扫描错误页改用 `QTableView + QAbstractTableModel`，移除“查看更多”分批灌表，降低大结果集 UI 控件开销。
- 可疑文件页新增“说明”和“建议”列，风险原因现在会给出可读解释和处理建议。
- QSS 表格样式同步覆盖 `QTableView`。
- 新增 PyInstaller 单文件打包配置 `FileAudit.spec`、`build_exe.ps1` 和 `build_exe_python312.ps1`，可生成无需安装 Python/PySide6 的 `dist\FileAudit.exe`。

## 2026-06-11

- 新增真实目录扫描核心，支持递归扫描、忽略目录、文件基础信息收集和错误记录。
- 新增扫描数据模型：`ScanOptions`、`FileRecord`、`DuplicateGroup`、`ScanSummary`、`ScanResult`。
- 新增后台扫描线程 `ScanWorker`，扫描过程不阻塞界面。
- 新增取消扫描功能，取消后保留已经扫描到的部分结果。
- 新增重复文件检测：先按文件大小分组，再对同大小文件计算 Hash。
- 新增风险检测：可疑扩展名、双扩展名、隐藏文件、空文件、大文件、时间异常和路径过长。
- 调整默认风险规则：`.js` 不再默认作为可疑扩展名。
- 新增扩展名白名单，白名单扩展名不触发可疑扩展名和双扩展名风险。
- 文件明细页接入真实扫描结果，支持搜索、风险筛选、文件类型筛选和表头排序。
- 重复文件页接入真实重复文件结果，显示完整路径、Hash 和可节省空间。
- 可疑文件页接入真实风险结果，并按高风险、中风险、低风险排序。
- 扫描概览页新增统计卡片和图表：文件类型分布、风险等级分布、目录占用 Top 10。
- 优化长路径显示，表格支持横向滚动并提供 tooltip。
- 优化滚动条样式，提高横向/纵向滚动条可见性。
- 新增 CSV 数据包和 HTML 图表报告导出。
- 新增设置页，支持默认目录、报告目录、阈值、Hash 算法、检测开关、忽略目录、可疑扩展名和白名单扩展名。
- 新增设置持久化，开发环境保存到项目根目录 `settings.json`，打包后保存到 exe 同级目录。
- 修正 `.gitignore`，只忽略根目录 `/reports/`，避免误忽略源码目录 `fileaudit/reports/`。
- 修复 Hash 阶段看似卡住的问题：底部状态栏现在会显示“正在计算重复文件 Hash”的阶段进度。
- 修复 Hash 大文件读取时取消不及时的问题：Hash 分块读取过程中也会检查取消请求。
- 降低超大扫描结果的 UI 内存压力：早期采用表格分批渲染，后续已改为 `QTableView + QAbstractTableModel`。
- 避免 PySide6 表格排序原生崩溃：不再自定义 `QTableWidgetItem.__lt__`，改为 Python 数据排序后重绘表格。
- 文档新增“已知问题与处理”“已遇到的问题和当前处理”“已经踩过的坑”等记录。
- 文档补充“忽略目录、可疑扩展名、白名单扩展名”的区别说明。
- 文档新增未来跳过规则规划：跳过指定目录/文件、隐藏文件和大文件处理方式、只跳过大文件 Hash 等。
