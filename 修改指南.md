# 修改指南

这份说明只解决一个问题：你想自己改东西时，应该打开哪个文件。

## 先记住这些地方

```text
main.py
```

程序入口。一般不用改，除非你要改启动方式。

```text
fileaudit/app.py
```

主窗口。负责页面组装、启动扫描、取消扫描、分发扫描结果、导出报告和同步设置。

```text
fileaudit/ui/pages.py
```

具体页面。扫描配置、扫描概览、文件明细、重复文件、可疑文件、报告导出、设置页都在这里。

```text
fileaudit/ui/components.py
```

通用组件。顶部栏、左侧导航、底部状态栏、统计卡片和图表控件在这里。

```text
fileaudit/ui/styles.py
```

界面样式。颜色、按钮、表格、滚动条、进度条等 QSS 都在这里。

```text
fileaudit/core/scanner.py
```

扫描核心。目录遍历、风险检测、Hash 和重复文件判断在这里。

```text
fileaudit/models/scan.py
```

数据结构。扫描配置、文件记录、重复文件组、扫描结果都在这里。

```text
fileaudit/reports/exporter.py
```

报告导出。CSV 和 HTML 报告生成逻辑在这里。

```text
fileaudit/config/settings.py
```

设置读写。默认设置、JSON 保存和加载在这里。


一个是窗口标题，一个是顶部栏标题。

### 改左侧菜单文字

打开：

```text

```

搜索：

```python
buttons = [
    ("扫描配置", 0),

    ...

```

只改双引号里的中文即可。后面的数字是页面编号，改页面顺序时才需要同步调整。

### 改扫描配置页

打开：

```text
fileaudit/ui/pages.py
```

搜索：

```python
class ScanConfigPage
```

这里可以改：

- 扫描目录输入框。
- 检测选项复选框。
- 大文件阈值。
- 路径长度阈值。
- Hash 算法。
- 忽略目录。
- 可疑扩展名。
- 白名单扩展名。

### 改扫描规则

打开：

```text
fileaudit/core/scanner.py
fileaudit/models/scan.py
```

常见位置：

```python
ScanOptions.suspicious_extensions
```

默认可疑扩展名列表在这里。

```python
_apply_risk_rules
```

风险规则在这里。比如要改隐藏文件、大文件、双扩展名的判断逻辑，就看这个函数。

当前几个设置的含义：

```text
忽略目录 = 整个目录不扫描
可疑扩展名 = 命中后标记风险
白名单扩展名 = 命中后不因扩展名标风险
```

如果你要做“跳过大文件”“跳过隐藏文件”，不要只改风险规则。那属于扫描前/扫描中的跳过规则，应该在 `scan_directory` 遍历文件时判断。

## 最容易改错的地方

- 不要让扫描核心 `fileaudit/core/` 直接引用 PySide6 控件。
- UI 更新要在主线程做，耗时扫描要走 `ScanWorker`。
- 左侧菜单编号要和 `app.py` 里页面添加顺序一致。
- 表格排序不要再自定义 `QTableWidgetItem.__lt__`，之前会触发 PySide6 原生崩溃。
- `.gitignore` 里不要写裸 `reports/`，否则会误忽略 `fileaudit/reports/` 源码目录。
- 中文文件保持 UTF-8 编码。
- 大目录扫描时不要把全部结果一次性灌入表格。当前用 `MAX_TABLE_ROWS = 5000` 控制 UI 渲染数量。
- 如果扫描时文件数不动，不一定是卡死，可能正在计算重复文件 Hash。看底部状态栏的阶段提示。

## 已经踩过的坑

### `.mui` 文件处看似卡住

扫 Windows 系统目录时，`.mui` 文件很多，且可能有大量同大小文件。文件遍历后会进入 Hash 阶段，此时“已扫描文件数”不会继续增加。

当前已经在底部状态栏显示：

```text
正在计算重复文件 Hash：x / y - 当前文件
```

### 内存占用过高

60 万文件级别如果全部显示到 `QTableWidget`，内存会非常高。当前只渲染前 5000 行，但报告导出仍使用完整数据。

如果后续要完整浏览大结果集，优先做分页或 `QAbstractTableModel`。

### 表格排序导致原生崩溃

不要用 Python 子类覆盖 `QTableWidgetItem.__lt__` 做排序。曾出现：

```text
引用 0x0000000000000018 内存，该内存不能为 read
```

当前安全做法是：点击表头后排序 Python 列表，再重绘表格。

## 推荐修改顺序

如果你刚开始自己改，建议按这个顺序练：

1. 先改页面文字。
2. 再改按钮文字。
3. 再改颜色、滚动条和字体大小。
4. 再改表格列。
5. 再改扫描规则。
6. 最后再改报告导出和设置持久化。

## 后续功能建议

### 跳过规则

可以考虑新增独立的跳过规则区域，和风险检测分开。

建议优先级：

1. 跳过超过指定大小的文件，例如大于 100 MB 或 1 GB。
2. 跳过隐藏文件。
3. 跳过指定文件名。
4. 跳过路径通配规则。
5. 只跳过大文件 Hash，不跳过文件记录。

这里要先想清楚：

```text
跳过文件 = 不出现在扫描结果里
跳过 Hash = 文件仍显示，但不参与重复文件检测
标记风险 = 文件显示，并给出风险原因
```
