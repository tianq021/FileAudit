# 修改指南

这份说明只解决一个问题：你想自己改东西时，应该打开哪个文件。

## 先记住这些地方

```text
settings.json
```

项目根目录下的真实设置文件。开发环境启动和保存设置都会读写这里；打包成 exe 后使用 exe 同级目录的 `settings.json`。不要再看 `%USERPROFILE%\.fileaudit\settings.json`，旧用户目录配置不会作为当前项目设置来源。

```text
main.py
```

程序入口。一般不用改，除非你要改启动方式。

```text
fileaudit/app.py
```

主窗口。负责页面组装、启动预检查、启动扫描、取消扫描、清空结果、分发扫描结果、导出报告和同步设置。

```text
fileaudit/ui/pages.py
```

具体页面。扫描配置、扫描概览、文件明细、重复文件、可疑文件、扫描错误、分析导出、设置页都在这里。

```text
fileaudit/ui/components.py
```

通用组件。顶部栏、左侧导航、底部状态栏、统计卡片、条形图和环形图在这里。

```text
fileaudit/ui/styles.py
```

界面样式。颜色、按钮、标签页、表格、滚动条、进度条等 QSS 都在这里。

```text
fileaudit/core/scanner.py
```

扫描核心。预检查、目录遍历、跳过规则、只扫描规则、风险检测、Hash 和重复文件判断在这里。

```text
fileaudit/models/scan.py
```

数据结构。扫描配置、文件记录、预检查结果、重复文件组、扫描错误、扫描摘要和扫描结果都在这里。

```text
fileaudit/reports/exporter.py
```

报告导出。CSV 和 HTML 报告生成逻辑、路径脱敏、报告图表和扫描错误预览在这里。

```text
fileaudit/config/settings.py
```

设置读写。默认设置、JSON 保存和加载在这里。

```text
fileaudit/utils/
```

公共工具。文件类型分类、大小/时间/风险格式化、跳过原因文案和图表分布统计在这里。

```text
tests/test_scanner.py
```

扫描核心和报告导出的测试。改扫描规则、跳过规则或报告导出时，优先补这里。

```text
tests/test_utils.py
```

公共工具测试。改 `fileaudit/utils/` 时优先补这里。

### 改扫描规则

打开：

```text
fileaudit/core/scanner.py
fileaudit/models/scan.py
```

常见位置：

```python
ScanOptions.suspicious_extensions
```

默认可疑扩展名列表在这里。

```python
_apply_risk_rules
```

风险规则在这里。比如要改隐藏文件、大文件、无扩展名大文件、临时文件、双扩展名的判断逻辑，就看这个函数。

```python
_file_skip_reason
_filter_dirs
```

跳过规则在这里。它们决定文件或目录是否完全不进入扫描结果。

默认跳过目录在这里：

```text
fileaudit/models/scan.py
DEFAULT_SKIP_DIRS
```

单文件超时保护在这里：

```text
fileaudit/core/scanner.py
_raise_if_timed_out
ScanOperationTimeout
```

```python
_file_matches_include_rules
```

只扫描规则在这里。开启 `include_only_matched` 后，未命中的文件会被跳过。

当前几个设置的含义：

```text
忽略目录 = 整个目录不扫描
跳过规则 = 文件或目录不进入扫描结果
只扫描规则 = 开启后只保留命中的文件
可疑扩展名 = 命中后标记风险
白名单扩展名 = 命中后不因扩展名标风险
隐藏文件处理 = 标记风险 / 跳过文件 / 忽略
大文件处理 = 标记风险 / 跳过文件 / 关闭，共用大文件阈值
```

## 最容易改错的地方

- 不要让扫描核心 `fileaudit/core/` 直接引用 PySide6 控件。
- UI 更新要在主线程做，耗时扫描要走 `ScanWorker`，预检查要走 `PreviewWorker`。
- 左侧菜单编号要和 `app.py` 里页面添加顺序一致。
- 新增设置字段时，要同步 `AppSettings`、`ScanOptions`、设置页、扫描配置页和保存逻辑。
- 不要在 UI 和报告里重复写格式化、分类和分布统计逻辑，优先放到 `fileaudit/utils/`。
- Windows 受保护目录优先放进 `DEFAULT_SKIP_DIRS`，不要靠捕获错误后再处理。
- 单文件超时不能强行中断已经卡死的系统调用，只能在调用返回或 Hash 分块读取间隙检查耗时。
- 表格排序不要再自定义 `QTableWidgetItem.__lt__`，之前会触发 PySide6 原生崩溃。
- `.gitignore` 里不要写裸 `reports/`，否则会误忽略 `fileaudit/reports/` 源码目录。
- 中文文件保持 UTF-8 编码。
- 大目录扫描时不要退回到一次性创建 `QTableWidgetItem`。结果页当前使用 `QTableView + QAbstractTableModel`，按需读取显示行。
- 如果扫描时文件数不动，不一定是卡死，可能正在计算重复文件 Hash。看底部状态栏的阶段、目录、跳过和错误提示。
- 跳过文件和标记风险不是一回事，不要把跳过逻辑写到 `_apply_risk_rules` 里。
- UI 上同一类设置尽量只保留一个入口。隐藏文件和大文件已经收敛为处理方式下拉框，不要再新增并列的检测/跳过开关。

## 已经踩过的坑

### `.mui` 文件处看似卡住

扫 Windows 系统目录时，`.mui` 文件很多，且可能有大量同大小文件。文件遍历后会进入 Hash 阶段，此时“已扫描文件数”不会继续增加。

当前已经在底部状态栏显示：

```text
正在计算重复文件 Hash：x / y，目录 n，跳过 n，错误 n - 当前文件
```

### 内存占用过高

60 万文件级别如果全部显示到 `QTableWidget`，内存会非常高。当前结果页已经改为 `QTableView + QAbstractTableModel`，避免一次性创建大量表格项，但报告导出和统计仍使用完整数据。

如果后续要进一步压低内存，优先把完整结果改成 SQLite 或分页数据源。

### 表格排序导致原生崩溃

不要用 Python 子类覆盖 `QTableWidgetItem.__lt__` 做排序。曾出现：

```text
引用 0x0000000000000018 内存，该内存不能为 read
```

当前安全做法是：点击表头后排序 Python 列表，再重置表格模型。

### 导出页复选框不生效

早期导出页有多个复选框，但实际导出固定生成完整报告包。当前已经改成说明文字，避免用户误解。以后如果要恢复复选框，需要先新增 `ExportOptions` 并接入真实导出逻辑。

## 推荐修改顺序

如果你刚开始自己改，建议按这个顺序练：

1. 先改页面文字。
2. 再改按钮文字。
3. 再改颜色、标签页、滚动条和字体大小。
4. 再改表格列。
5. 再改风险规则。
6. 再改跳过规则或只扫描规则。
7. 最后再改报告导出和设置持久化。
