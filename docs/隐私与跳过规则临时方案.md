# 隐私与跳过规则临时方案

这份文件用于记录 FileAudit 的隐私保护、跳过规则、只扫描规则和报告路径脱敏设计。

## 当前风险

设置文件固定在项目根目录 `settings.json`。开发环境只读写这份文件；打包后读写 exe 同级目录 `settings.json`。不要再依赖 `%USERPROFILE%\.fileaudit\settings.json`，避免旧用户目录配置里的只扫描规则或跳过规则影响当前项目。

- 文件明细默认会显示完整路径。
- 报告导出默认会写入完整路径。
- 如果未配置跳过规则，隐藏文件、私密目录、聊天软件目录、浏览器数据目录、证书密钥文件等可能进入结果。
- 白名单扩展名只是不按扩展名判风险，不代表跳过或隐藏。

## 概念区分

```text
跳过 = 不扫描、不显示、不导出
只扫描 = 开启后，仅保留命中规则的文件
白名单 = 扫描并显示，但不因扩展名判风险
脱敏 = 扫描并参与统计，但导出时隐藏或替换路径/文件名
```

## 当前已经实现

### 跳过规则

当前支持：

- 跳过指定目录名。
- 跳过指定文件名。
- 跳过指定扩展名。
- 跳过路径关键词。

隐藏文件和大文件不再以独立跳过开关展示，而是在设置中各自使用一个处理方式：

```text
隐藏文件处理 = 标记风险 / 跳过文件 / 忽略
大文件处理 = 标记风险 / 跳过文件 / 关闭
大文件阈值 = 标记或跳过大文件共用同一个 MB 阈值
```

选择“跳过文件”时，文件不会生成 `FileRecord`，也不会进入界面表格和导出报告。选择“标记风险”时，文件会保留在结果中并带风险原因。

默认跳过扩展名包含：

```text
.key
.pem
.pfx
.kdbx
.wallet
```

默认跳过目录包含：

```text
System Volume Information
$RECYCLE.BIN
Recovery
Config.Msi
```

这些目录通常是 Windows 系统、回收站、恢复或安装事务目录，整盘扫描时很容易出现 `WinError 5` 拒绝访问。默认跳过可以减少无意义的扫描错误。

扫描流程中会在创建 `FileRecord` 之前判断是否跳过：

```text
遍历目录
  -> 判断目录是否跳过
  -> 判断文件是否跳过
  -> 未跳过才生成 FileRecord
```

### 只扫描规则

当前支持开启“只扫描匹配规则的文件”。命中任一规则即可进入结果：

- 扩展名。
- 文件名关键词。
- 路径关键词。
- 文件类型，例如 `图片`、`文档`、`音视频`、`压缩包`、`代码`、`可执行`。

跳过规则固定优先于只扫描规则。文件即使命中只扫描规则，只要同时命中跳过规则，仍然会被跳过。UI 不再提供规则优先级下拉框，避免隐私跳过规则被另一个设置覆盖。

### 规则冲突提示

扫描配置页和设置页会实时显示规则状态，提前提示常见冲突：

- 开启“只扫描匹配规则”但没有填写任何只扫描规则。
- 可疑扩展名同时出现在白名单扩展名中。
- 可疑扩展名同时出现在跳过扩展名中。
- 只扫描路径关键词落入已经被忽略或跳过的目录。
- 只扫描路径关键词和跳过路径关键词互相包含。

实时提示复用保存/扫描前的同一套校验逻辑，最终点击“开始扫描”或“保存设置”时仍会再次拦截。

### 扫描前预检查

扫描配置页新增“预检查”。预检查会应用跳过规则和只扫描规则，并估算：

- 预计进入结果的文件数量。
- 实际访问的目录数量。
- 跳过文件和跳过目录数量。
- 跳过原因。
- 开启重复检测时可能进入 Hash 阶段的候选文件数。

预检查不会生成完整 `FileRecord`，也不会计算 Hash。它适合在正式扫描前确认隐私跳过规则是否生效，尤其适合整盘扫描、大目录扫描或刚修改规则之后。

### 跳过统计

`ScanSummary` 当前包含：

```python
skipped_files: int
skipped_dirs: int
skip_reasons: dict[str, int]
```

概览页和报告可以显示：

```text
已跳过文件：123
已跳过目录：8
跳过原因：隐藏文件 20，大文件 3，路径关键词 100
```

### 报告路径脱敏

设置页新增“报告导出完整路径”开关。

- 开启：CSV 和 HTML 报告写完整路径。
- 关闭：CSV 和 HTML 报告只写文件名或目录名。

注意：这个开关只影响导出报告，不影响界面显示和扫描结果本身。

### 权限错误提示

当前遇到 `PermissionError` 或 Windows `WinError 5` 时，会把原始错误转换成更易懂的提示：

```text
无权限访问。Windows 受保护目录或系统文件通常会出现这个提示，可以在跳过目录中排除。
```

## 建议配置

如果要降低私密文件进入结果的概率，可以在设置页增加：

```text
跳过目录名：
AppData
WeChat Files
Tencent Files
Chrome
Edge

跳过扩展名：
.key
.pem
.pfx
.kdbx
.wallet

跳过路径关键词：
\WeChat Files\
\Tencent Files\
\Chrome\User Data\
\Edge\User Data\
```

如果只想扫描某一类文件，可以开启“只扫描匹配规则”，例如：

```text
只扫描扩展名：
.pdf
.docx
.xlsx

只扫描文件类型：
文档
图片
```

## 后续可选优化

### 界面脱敏显示

当前界面仍显示完整路径。后续可以增加界面脱敏开关：

- 只显示文件名。
- 只显示相对路径。
- 用户名替换成 `<USER>`。
- 表格 tooltip 不显示完整路径。

### 更强的路径匹配

当前路径关键词是简单包含匹配。后续可以支持：

- 通配符。
- 正则表达式。
- Windows/Linux 路径分隔符自动兼容。
- 规则预览和命中测试。

### 只跳过 Hash

当前“大文件处理：跳过文件”会让文件完全不进入扫描结果。后续可以增加“只跳过大文件 Hash”：

```text
文件仍进入明细
文件仍参与大小统计和风险统计
文件不参与重复文件完整 Hash
```

这样更适合视频、虚拟机镜像、压缩包等超大文件场景。
