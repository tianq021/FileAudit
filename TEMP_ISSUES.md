# FileAudit 暂时问题记录

> 更新时间：2026-06-13  
> 文件用途：临时记录当前项目中已经发现的问题、风险点和后续优化方向。

## 1. 当前项目状态

设置文件已经固定为项目根目录 `settings.json`。开发环境读写这份文件；打包后读写 exe 同级目录 `settings.json`。旧的 `%USERPROFILE%\.fileaudit\settings.json` 不再作为配置来源，避免历史“只扫描 .log”等旧规则影响当前扫描。

FileAudit 当前已经完成第一版主体功能，并补上了一批第一版体验问题：

- 能启动 PySide6 桌面窗口。
- 能打包成 Windows 单文件 `dist\FileAudit.exe`，普通用户可双击启动。
- `dist` 中已保留同级 `settings.json`，方便打包版保存和读取设置。
- 能选择扫描目录。
- 能递归扫描文件。
- 能后台扫描，不阻塞界面。
- 能取消扫描，并保留部分结果。
- 能清空扫描结果。
- 能记录文件基础信息。
- 能通过“大小相同 + Hash 相同”判断重复文件。
- 能检测双扩展名、脚本文件、可疑扩展名、隐藏文件、空文件、大文件、无扩展名大文件、临时/备份文件、时间异常、路径过长。
- 能通过“隐藏文件处理”和“大文件处理”选择标记风险、跳过或忽略，并能跳过指定目录、指定文件名、指定扩展名和路径关键词。
- 能开启“只扫描匹配规则”。
- 能在规则变更时实时提示配置冲突，提前发现跳过规则、只扫描规则和检测规则之间的矛盾。
- 能在正式扫描前执行“预检查”，估算扫描规模、跳过情况和 Hash 候选文件数。
- 能统计跳过文件、跳过目录和跳过原因。
- 大目录扫描时能在底部状态栏显示文件数、目录数、跳过数、错误数和当前路径。
- 结果表格已改为 `QTableView + QAbstractTableModel`，减少大结果集 UI 控件开销。
- 可疑文件页能显示风险原因说明和建议处理动作。
- 能在界面中显示扫描概览、文件明细、重复文件、可疑文件和扫描错误。
- 概览和 HTML 报告已包含风险目录、重复扩展名和扫描错误目录等定位型图表。
- 修改时间分布的月份阈值已进入设置，默认 3 个月。
- 单文件超时阈值已进入设置，默认 15 秒；慢文件和慢 Hash 会被记录后跳过。
- 隐藏文件和大文件相关设置已收敛为单一处理方式，避免同时配置“检测”和“跳过”。
- 能导出 CSV 数据包和 HTML 报告。
- 能选择报告是否导出完整路径。
- 默认跳过 Windows 常见受保护目录，减少整盘扫描时的拒绝访问噪音。
- 权限错误会显示更友好的“无权限访问”说明。
- 能保存默认扫描目录、报告目录和扫描规则。
- 已新增扫描核心、报告导出和公共工具测试。

当前主要问题已经从“第一版功能是否闭合”转向：

```text
大数据量体验
报告能力增强
页面拆分维护性
更细的隐私脱敏
历史记录和对比分析
```

## 2. 已解决的问题

### 2.1 README 的运行方式为空

已处理：README 已补充普通用户和开发者两种运行方式。普通用户拿到 `FileAudit.exe` 和 `settings.json` 后可直接双击启动；当前机器开发者运行源码和打包优先使用 `C:\Python312\python.exe`，不要默认使用项目里的 `.venv`，因为 `.venv` 不能假定已经安装依赖。

同时新增 `发布说明.md`，用于朋友圈或普通用户转发，重点说明启动方式、第一次怎么扫描和隐私提醒。

### 2.2 “清空结果”按钮没有接功能

已处理：`ScanConfigPage` 新增 `clear_requested` 信号，`MainWindow.on_clear_requested` 会清空各结果页并重置状态栏。

### 2.3 报告导出页的复选框和真实导出逻辑不一致

已处理：暂未生效的复选框已移除，导出页改为说明当前会生成完整报告包。

### 2.4 设置页数字输入缺少完整校验

已处理：设置页和扫描配置页都增加了 `QIntValidator` 和 `parse_positive_int`，错误提示会指出具体字段。

### 2.5 扫描配置页错误提示不够准确

已处理：`MainWindow.on_scan_requested` 会展示具体 `ValueError` 文案。

### 2.6 缺少扫描核心测试

已处理：新增 `tests/test_scanner.py`，覆盖风险规则、重复检测、跳过规则、只扫描规则、报告路径脱敏和扫描错误导出。

### 2.7 缺少无扩展名大文件规则

已处理：大文件且无扩展名时标记 `large file without extension`，风险等级为中风险。

### 2.8 缺少临时文件、缓存文件、备份文件识别

已处理：`.tmp`、`.temp`、`.cache`、`.bak`、`.old` 和 `~` 结尾文件会标记为临时/备份文件，风险等级为低风险。

## 3. 高优先级问题

### 2.16 大目录表格性能仍有优化空间

已处理：

- 文件明细、重复文件、可疑文件和扫描错误页已改为 `QTableView + QAbstractTableModel`。
- 结果页不再一次性创建大量 `QTableWidgetItem`。
- “查看更多”分批灌表按钮已移除。

剩余风险：

- 完整 `ScanResult` 仍常驻内存，用于统计、筛选和导出。
- 超大结果集的筛选和排序仍然基于内存列表。

后续如果继续优化，应考虑：

- SQLite 或分页数据源。
- 把筛选、排序和导出迁移到分页数据源。

### 2.9 UI 和报告中存在重复格式化函数

已处理：新增 `fileaudit/utils/`，把这些函数集中到公共模块：

- `format_size`
- `format_datetime`
- `format_risk_level`
- `format_risk_reasons`
- `classify_file_type`
- `format_skip_reason`
- `build_*_distribution`

并新增 `tests/test_utils.py` 保护这些公共逻辑。

### 2.10 Windows 受保护目录拒绝访问提示太生硬

已处理：

- 默认跳过 `System Volume Information`、`$RECYCLE.BIN`、`Recovery`、`Config.Msi`。
- `PermissionError` / `WinError 5` 会显示成“无权限访问”说明。
- 新增测试覆盖默认跳过目录和权限错误文案。

### 2.11 单个文件处理过慢会拖住扫描

已处理：

- 新增 `file_timeout_seconds`，默认 15 秒，`0` 表示关闭。
- 文件元数据处理超过阈值时记录扫描错误，并计入“慢文件”跳过原因。
- Hash 计算超过阈值时记录扫描错误，并计入“慢 Hash”跳过原因。
- 新增测试覆盖超时文案、默认设置和跳过原因显示。

注意：这是协作式超时保护，不能强行打断已经卡在操作系统内部的单次 `stat()` 或 `read()` 调用。

### 2.12 设置项容易互相冲突

已处理：

- 隐藏文件不再同时展示检测和跳过两个开关，改为“隐藏文件处理：标记风险 / 跳过文件 / 忽略”。
- 大文件不再同时展示检测阈值和跳过阈值，改为“大文件处理：标记风险 / 跳过文件 / 关闭”，共用“大文件阈值”。
- 跳过规则固定优先于只扫描规则，界面不再提供规则优先级下拉框。
- 新增 `fileaudit/config/validation.py`，拦截可疑扩展名与白名单/跳过扩展名重叠、只扫描规则为空、只扫描路径落入跳过路径等配置。
- 新增 `tests/test_ui_settings.py` 覆盖设置冲突校验。

### 2.13 规则冲突提示不够前置

已处理：

- 扫描配置页和设置页新增“规则状态：正常 / 有冲突”提示。
- 检测开关、处理模式、阈值、跳过规则和只扫描规则变更时会自动刷新校验结果。
- 页面提示复用 `validate_detection_skip_conflicts()`，开始扫描和保存设置时仍保留最终拦截。

### 2.14 大目录扫描前缺少规模预估

已处理：

- 扫描配置页新增“预检查”按钮。
- 新增 `ScanPreview`、`preview_scan()` 和 `PreviewWorker`。
- 预检查会估算预计扫描文件数、目录数、总大小、跳过文件/目录数量、跳过原因和 Hash 候选文件数。
- 预检查复用正式扫描的跳过规则和只扫描规则，但不生成完整 `FileRecord`，也不计算 Hash。
- 新增测试覆盖预检查统计。

### 2.15 大目录扫描过程反馈不足

已处理：

- 正式扫描新增详细进度回调。
- 底部状态栏显示已扫描文件数、已访问目录数、已跳过数量、错误数和当前路径。
- Hash 阶段继续显示 `当前 / 总数`，并保留目录、跳过和错误数量。
- 超长路径在底部状态栏中会被缩短显示。
- 新增测试覆盖详细扫描进度回调。

## 4. 中优先级问题

### 4.1 风险等级规则还可以更细

**当前情况：**

风险等级已经拆成高、中、低，但规则仍然比较粗。

**建议处理：**

- 高风险：双扩展名伪装。
- 中风险：脚本文件、路径异常、时间异常、隐藏文件、无扩展名大文件。
- 低风险：普通安装包、临时文件、空文件、大文件。
- 可疑文件页已新增风险解释和建议处理动作；后续可继续细化风险等级和建议文案。

**优先级：中**

### 4.2 Hash 阶段可以增加大文件跳过策略

**问题说明：**

当前重复文件检测会对同大小文件计算完整 Hash。如果扫描大量视频、压缩包、虚拟机镜像，Hash 阶段可能比较慢。

**建议处理：**

增加选项：

```text
只跳过超过指定大小的文件 Hash
```

注意这不同于“大文件处理：跳过文件”。后者会让文件完全不进入扫描结果。

**优先级：中**

### 4.3 缺少快速 Hash 策略

**问题说明：**

当前重复文件判断是：

```text
大小相同 -> 完整 Hash
```

**建议优化：**

未来可以改成：

```text
大小相同 -> 快速 Hash -> 完整 Hash
```

快速 Hash 可以读取：

- 文件头 1MB。
- 文件尾 1MB。
- 文件大小。

**优先级：中**

### 4.4 缺少 Excel 多 Sheet 报告

**问题说明：**

当前导出是 CSV 数据包 + HTML 报告，还没有 Excel 多 Sheet 报告。

**建议处理：**

新增：

```text
FileAudit_报告_时间戳.xlsx
```

Sheet 建议：

- 扫描概览。
- 文件明细。
- 重复文件。
- 可疑文件。
- 扫描错误。
- 跳过统计。

**优先级：中**

### 4.5 界面路径脱敏还没做

**当前情况：**

报告导出可以关闭完整路径，但界面仍然显示完整路径。

**建议处理：**

增加界面显示策略：

```text
显示完整路径
只显示文件名
只显示相对路径
用户名替换成 <USER>
```

**优先级：中**

## 5. 功能缺口

### 5.1 缺少独立目录分析页

当前扫描概览里已经有目录占用 Top 10，但还没有独立目录分析页面。

建议页面字段：

- 目录路径。
- 文件数量。
- 总大小。
- 可疑文件数量。
- 重复文件数量。
- 最大文件。

**优先级：中**

### 5.2 缺少独立时间分析页

当前已经能检测未来修改时间，并在概览中有修改时间分布，但还没有完整时间分析页面。

建议功能：

- 最近 1 天修改文件。
- 最近 7 天修改文件。
- 未来时间文件。
- 创建时间很新但修改时间很旧的文件。
- 长期未访问文件。

**优先级：中**

### 5.3 缺少扫描历史记录

当前扫描结果只存在当前运行中，关闭程序后无法查看历史。

建议后续把扫描摘要保存为 JSON 或 SQLite。

**优先级：低**

### 5.4 缺少两次扫描结果对比

目前只能看单次扫描结果。

建议后续支持：

- 新增文件。
- 删除文件。
- 修改文件。
- 新增风险文件。
- 新增重复文件。

**优先级：低**

## 6. 建议修改顺序

### 第一阶段：稳住第一版

1. 运行并修复现有测试。
2. 补充更多跳过规则和只扫描规则测试。
3. 拆分 `pages.py`。
4. 给 `fileaudit/utils/` 继续补边界测试。

### 第二阶段：增强报告能力

1. 增加 Excel 多 Sheet 导出。
2. 优化 HTML 报告内容。
3. 报告中复用风险解释和处理建议。
4. 支持打开报告目录、复制报告路径。

### 第三阶段：增加分析页面

1. 增加目录分析页。
2. 增加时间分析页。
3. 增加文件清理建议。
4. 增加文件清理/隔离操作入口。

### 第四阶段：优化大数据量性能

1. 超大扫描结果改为 SQLite 或分页数据源。
2. Hash 增加只跳过大文件 Hash 策略。
3. 增加快速 Hash。
4. 增加扫描结果历史保存。
5. 增加两次扫描结果对比。
