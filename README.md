# FileAudit 文件体检分析器

FileAudit 是一个基于 PySide6 的桌面文件审计工具，用于扫描文件夹并分析文件基础信息、重复文件、可疑文件、隐私跳过规则、扫描错误和目录占用情况，并支持导出 CSV 数据包和 HTML 图表报告。

## 当前功能

- 选择目录并递归扫描文件，后台运行，不阻塞界面。
- 支持取消扫描，取消后保留已经扫描到的部分结果。
- 支持清空当前扫描结果，恢复未扫描状态。
- 支持扫描前“预检查”，可先估算预计扫描文件数、目录数、总大小、跳过数量、跳过原因和 Hash 候选文件数。
- 扫描配置页和设置页会实时显示规则状态，提前提示跳过规则、只扫描规则和检测规则之间的冲突。
- 大目录扫描时底部状态栏会显示已扫描文件数、已访问目录数、已跳过数量、错误数和当前路径。
- 记录文件名、完整路径、扩展名、大小、创建时间、修改时间、Hash、隐藏状态和空文件状态。
- 按“大小相同 + Hash 相同”识别重复文件。
- 检测双扩展名伪装、脚本文件、可疑扩展名、隐藏文件、空文件、大文件、无扩展名大文件、临时/备份文件、时间异常和路径过长。
- 文件明细支持关键字搜索、风险等级筛选、文件类型筛选和表头排序。
- 文件明细、重复文件、可疑文件和扫描错误结果表格使用 `QTableView + QAbstractTableModel`，按可见区域读取显示行，减少大结果集 UI 内存压力。
- 可疑文件页显示风险原因、原因说明和建议处理动作，便于判断下一步是隔离、清理、归档还是修正路径/时间。
- 扫描概览显示统计卡片和图表：文件类型、风险等级、文件大小、风险原因、风险目录、目录占用、扩展名、最大文件、重复文件组、重复扩展名、修改时间、跳过原因和扫描错误目录。
- 重复文件页、可疑文件页和扫描错误页显示完整路径，支持横向滚动查看长路径。
- 报告导出支持 `summary.csv`、`files.csv`、`duplicates.csv`、`risks.csv`、`errors.csv` 和 `report.html`。
- 设置页支持保存默认目录、阈值、检测开关、隐藏文件处理方式、大文件处理方式、忽略目录、可疑扩展名、白名单扩展名、跳过规则、只扫描规则和报告是否导出完整路径。
- 鼠标悬停在扫描配置、规则输入框和设置项上时会显示解释，说明该功能的影响和匹配方式。
- 设置页支持配置“修改时间分布”的月份阈值，默认 3 个月。
- 设置页支持配置单文件处理超时，默认 15 秒，超时文件会记录后跳过，继续扫描后续文件。
- 扫描核心有 `tests/test_scanner.py` 覆盖风险检测、重复文件、跳过规则、只扫描规则和报告脱敏导出。

## 运行方式

```bash
git clone https://github.com/tianq021/FileAudit.git
cd FileAudit

python -m venv .venv
.venv\Scripts\activate

pip install -r requirements.txt
python main.py
```

如果已经在项目目录中，也可以直接执行：

```bash
pip install -r requirements.txt
python main.py
```

```bash
python -m unittest discover tests
```

## 生成 Windows 单文件 exe

最终用户不需要安装 Python、PySide6 或其他依赖。开发者在 Windows 上执行：

```powershell
.\build_exe.ps1 -Clean
```

脚本会创建独立的 `.build-venv`，安装 `requirements.txt` 和 PyInstaller，然后生成：

```text
dist\FileAudit.exe
```

把这个 `FileAudit.exe` 发给用户即可双击运行。首次启动单文件版时会比普通目录版慢一些，这是 PyInstaller 解包运行的正常现象。

如果本机已有 `C:\Python312`，且其中已经安装 PySide6 和 PyInstaller，也可以使用：

```powershell
powershell -ExecutionPolicy Bypass -File .\build_exe_python312.ps1
```

当前已验证该脚本可以生成 `dist\FileAudit.exe`。

## 使用流程

1. 打开程序。
2. 在“扫描配置”页选择扫描目录。
3. 按需选择“全部扫描”或“只扫描匹配规则”，调整基础检测选项。
4. 在“设置”页配置隐藏文件处理、大文件处理、忽略目录、跳过规则、只扫描规则和导出路径策略。
5. 查看页面底部的“规则状态”，如有冲突先按提示修正。
6. 可点击“预检查”先估算扫描规模、跳过情况和 Hash 候选文件数。
7. 点击“开始扫描”。
8. 扫描过程中通过底部状态栏查看文件数、目录数、跳过数、错误数和当前路径。
9. 扫描完成后查看“扫描概览”“文件明细”“重复文件”“可疑文件”“扫描错误”。
10. 在“分析+导出”页点击“生成报告”，选择输出目录。

## 规则概念

```text
忽略目录 = 目录名命中后，整个目录不扫描
跳过规则 = 文件或目录不进入扫描结果，也不会出现在报告中
只扫描规则 = 开启后，只有命中规则的文件才进入扫描结果
风险规则 = 文件进入结果，但被标记为高/中/低风险
白名单扩展名 = 文件进入结果，但不因指定扩展名被标记风险
导出脱敏 = 文件参与扫描和统计，但报告中不写完整路径
```

扫描配置页和设置页会在规则变更时自动校验常见冲突，例如：

- 可疑扩展名同时出现在白名单或跳过扩展名中。
- 开启“只扫描匹配规则”但没有填写任何只扫描规则。
- 只扫描路径关键词落入已经被跳过的目录或路径关键词。

有冲突时页面会直接显示“规则状态：有冲突”，开始扫描和保存设置时也会再次拦截。

## 扫描前预检查

“预检查”会快速遍历目录并应用跳过规则、只扫描规则，但不会生成完整文件记录，也不会计算 Hash。它用于在正式扫描前判断：

- 预计会进入结果的文件数量。
- 实际访问的目录数量。
- 预计扫描文件总大小。
- 会被跳过的文件和目录数量。
- 主要跳过原因。
- 开启重复检测时，可能进入 Hash 阶段的候选文件数。

预检查适合在扫描大目录、整盘目录或规则刚调整完时使用。

## 重复文件判断

重复文件不是只看文件名，而是：

1. 先按文件大小分组。
2. 只有同样大小的文件才计算 Hash。
3. 大小相同且 Hash 相同，才判定为重复文件。

这样可以避免给所有文件无差别计算 Hash，扫描大目录时更快。

## 风险规则

默认风险规则包括：

- 高风险：双扩展名伪装，例如 `invoice.pdf.exe`。
- 中风险：脚本文件、隐藏文件、时间异常、路径过长、无扩展名大文件。
- 低风险：普通可疑扩展名、临时/备份文件、空文件、大文件。

`.js` 默认不再作为可疑扩展名，但仍归类为代码文件。可疑扩展名和白名单扩展名都可以在设置页修改。

可疑文件页会同时展示：

- 原因：命中了哪些风险规则。
- 说明：为什么这个规则值得关注。
- 建议：建议先确认来源、隔离、清理、归档或修正路径/时间等操作。

## 隐藏文件和大文件处理

隐藏文件和大文件各自只保留一个处理入口，避免同时出现“检测”和“跳过”的冲突设置：

```text
隐藏文件处理 = 标记风险 / 跳过文件 / 忽略
大文件处理 = 标记风险 / 跳过文件 / 关闭
大文件阈值 = 标记或跳过大文件共用同一个 MB 阈值
```

选择“标记风险”时，文件会进入结果并带风险标签。选择“跳过文件”时，文件不会进入扫描结果，也不会出现在导出报告中。

## 跳过规则

跳过规则表示“不要扫描这些内容”。当前支持：

- 跳过指定目录名。
- 跳过指定文件名。
- 跳过指定扩展名。
- 跳过路径关键词。

隐藏文件和大文件的跳过不再作为独立开关展示，分别由“隐藏文件处理”和“大文件处理”控制。

默认跳过目录包括：

```text
System Volume Information
$RECYCLE.BIN
Recovery
Config.Msi
```

跳过的文件不会进入内存中的扫描结果，也不会出现在界面表格和导出报告中。概览和报告会统计跳过文件数、跳过目录数和跳过原因。

## 超时保护

为避免扫描长时间停在单个文件上，当前支持“单文件超时”设置：

```text
默认：15 秒
0：关闭超时保护
```

如果读取文件元数据或计算 Hash 超过阈值，程序会：

- 在扫描错误页记录该文件。
- 在跳过原因里记录“慢文件”或“慢 Hash”。
- 继续扫描后续文件。

注意：如果操作系统底层调用本身长时间不返回，程序只能在这次调用返回后判断超时；普通 Python 代码不能强行中断已经卡死的系统调用。

## 只扫描规则

开启“只扫描匹配规则”后，只有命中以下任一规则的文件才会进入结果：

- 扩展名。
- 文件名关键词。
- 路径关键词。
- 文件类型，例如 `图片`、`文档`、`代码`、`可执行`。

跳过规则固定优先于只扫描规则。也就是说，文件即使命中只扫描规则，只要同时命中跳过规则，仍会被跳过。

## 报告导出

报告导出会生成一个带时间戳的目录，包含：

```text
summary.csv      # 扫描概览
files.csv        # 文件明细
duplicates.csv   # 重复文件
risks.csv        # 可疑文件
errors.csv       # 扫描错误
report.html      # 带图表的 HTML 报告
```

CSV 使用 `utf-8-sig` 编码，方便用 Excel 打开中文内容。设置页可以关闭“报告导出完整路径”，关闭后 CSV 和 HTML 报告只写文件名或目录名，降低完整路径泄露风险。

## 设置文件

开发环境下，设置保存到项目根目录：

```text
settings.json
```

项目根目录已经包含一份默认 `settings.json`。程序启动和保存设置都读写这份文件，不再读取 `%USERPROFILE%\.fileaudit\settings.json`。

设置页会把位置显示为项目内相对路径 `settings.json`，鼠标悬停可查看实际读写的完整路径。扫描时如果配置了“跳过路径关键词”或“只扫描路径关键词”，匹配对象是文件/目录的完整路径字符串。

设置页还支持：

- `另存为`：把当前页面设置导出为另一份 JSON 文件。
- `添加`：从已有 JSON 设置文件载入到当前页面；确认无误后点击“保存设置”，会写回当前项目的 `settings.json`。

打包成 exe 后，设置保存到 exe 同级目录的 `settings.json`。

如果设置文件损坏，程序会自动回退到默认设置。

## 时间图表

扫描概览和 HTML 报告里的“修改时间分布”默认按以下范围分类：

```text
今天
7 天内
30 天内
3 个月内
更早
未来时间
```

其中“3 个月内”可以在设置页的“修改时间分类”中调整。

## 已知问题与处理

### 大目录扫描内存较高

程序会保留完整扫描结果用于统计、筛选和报告导出。为降低 UI 表格控件开销，文件明细、重复文件、可疑文件和扫描错误表格已经改为 `QTableView + QAbstractTableModel`，视图按需读取可见行，不再一次性创建大量 `QTableWidgetItem`。

后续如果继续优化，应减少完整结果常驻内存，或把扫描结果保存到 SQLite/分页数据源。

### Windows 目录看似卡住

扫描 `C:\Windows` 时，可能在 `.mui` 等系统资源文件处停留较久。原因通常不是目录遍历卡死，而是进入了重复文件 Hash 阶段。

当前底部状态栏会区分“正在扫描”和“正在计算重复文件 Hash”，扫描阶段会显示文件数、目录数、跳过数、错误数和当前路径；Hash 阶段会显示 `当前 / 总数`、目录数、跳过数、错误数和当前处理文件。Hash 读取过程中也会检查取消请求。

### 受保护目录拒绝访问

Windows 的 `System Volume Information`、`$RECYCLE.BIN`、`Recovery` 等目录通常需要系统权限。当前默认会跳过这些目录；如果仍遇到 `WinError 5`，扫描错误页会显示更友好的“无权限访问”提示。

### 表格排序崩溃

开发过程中尝试过自定义 `QTableWidgetItem.__lt__` 来做真实大小排序，但 PySide6 在原生排序时触发过进程级内存读取错误。

当前不再启用 Qt 内置排序。文件明细页点击表头时，由 Python 对数据列表排序后重新绘制表格。
