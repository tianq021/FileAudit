# FileAudit 文件体检分析器

FileAudit 是一个基于 PySide6 的桌面文件审计工具，用于扫描文件夹并分析文件基础信息、重复文件、可疑文件、隐私跳过规则、扫描错误和目录占用情况，并支持导出 CSV 数据包和 HTML 图表报告。

## 当前功能

- 选择目录并递归扫描文件，后台运行，不阻塞界面。
- 支持取消扫描，取消后保留已经扫描到的部分结果。
- 支持清空当前扫描结果，恢复未扫描状态。
- 记录文件名、完整路径、扩展名、大小、创建时间、修改时间、Hash、隐藏状态和空文件状态。
- 按“大小相同 + Hash 相同”识别重复文件。
- 检测双扩展名伪装、脚本文件、可疑扩展名、隐藏文件、空文件、大文件、无扩展名大文件、临时/备份文件、时间异常和路径过长。
- 文件明细支持关键字搜索、风险等级筛选、文件类型筛选和表头排序。
- 扫描概览显示统计卡片和图表：文件类型、风险等级、文件大小、风险原因、风险目录、目录占用、扩展名、最大文件、重复文件组、重复扩展名、修改时间、跳过原因和扫描错误目录。
- 重复文件页、可疑文件页和扫描错误页显示完整路径，支持横向滚动查看长路径。
- 报告导出支持 `summary.csv`、`files.csv`、`duplicates.csv`、`risks.csv`、`errors.csv` 和 `report.html`。
- 设置页支持保存默认目录、阈值、检测开关、隐藏文件处理方式、大文件处理方式、忽略目录、可疑扩展名、白名单扩展名、跳过规则、只扫描规则和报告是否导出完整路径。
- 设置页支持配置“修改时间分布”的月份阈值，默认 3 个月。
- 设置页支持配置单文件处理超时，默认 15 秒，超时文件会记录后跳过，继续扫描后续文件。
- 扫描核心有 `tests/test_scanner.py` 覆盖风险检测、重复文件、跳过规则、只扫描规则和报告脱敏导出。

## 运行方式

```bash
git clone https://github.com/tianq021/FileAudit.git
cd FileAudit

python -m venv .venv
.venv\Scripts\activate

pip install -r requirements.txt
python main.py
```

如果已经在项目目录中，也可以直接执行：

```bash
pip install -r requirements.txt
python main.py
```

```bash
python -m unittest discover tests
```

## 使用流程

1. 打开程序。
2. 在“扫描配置”页选择扫描目录。
3. 按需选择“全部扫描”或“只扫描匹配规则”，调整基础检测选项。
4. 在“设置”页配置隐藏文件处理、大文件处理、忽略目录、跳过规则、只扫描规则和导出路径策略。
5. 点击“开始扫描”。
6. 扫描完成后查看“扫描概览”“文件明细”“重复文件”“可疑文件”“扫描错误”。
7. 在“分析+导出”页点击“生成报告”，选择输出目录。

## 规则概念

```text
忽略目录 = 目录名命中后，整个目录不扫描
跳过规则 = 文件或目录不进入扫描结果，也不会出现在报告中
只扫描规则 = 开启后，只有命中规则的文件才进入扫描结果
风险规则 = 文件进入结果，但被标记为高/中/低风险
白名单扩展名 = 文件进入结果，但不因指定扩展名被标记风险
导出脱敏 = 文件参与扫描和统计，但报告中不写完整路径
```

## 重复文件判断

重复文件不是只看文件名，而是：

1. 先按文件大小分组。
2. 只有同样大小的文件才计算 Hash。
3. 大小相同且 Hash 相同，才判定为重复文件。

这样可以避免给所有文件无差别计算 Hash，扫描大目录时更快。

## 风险规则

默认风险规则包括：

- 高风险：双扩展名伪装，例如 `invoice.pdf.exe`。
- 中风险：脚本文件、隐藏文件、时间异常、路径过长、无扩展名大文件。
- 低风险：普通可疑扩展名、临时/备份文件、空文件、大文件。

`.js` 默认不再作为可疑扩展名，但仍归类为代码文件。可疑扩展名和白名单扩展名都可以在设置页修改。

## 隐藏文件和大文件处理

隐藏文件和大文件各自只保留一个处理入口，避免同时出现“检测”和“跳过”的冲突设置：

```text
隐藏文件处理 = 标记风险 / 跳过文件 / 忽略
大文件处理 = 标记风险 / 跳过文件 / 关闭
大文件阈值 = 标记或跳过大文件共用同一个 MB 阈值
```

选择“标记风险”时，文件会进入结果并带风险标签。选择“跳过文件”时，文件不会进入扫描结果，也不会出现在导出报告中。

## 跳过规则

跳过规则表示“不要扫描这些内容”。当前支持：

- 跳过指定目录名。
- 跳过指定文件名。
- 跳过指定扩展名。
- 跳过路径关键词。

隐藏文件和大文件的跳过不再作为独立开关展示，分别由“隐藏文件处理”和“大文件处理”控制。

默认跳过目录包括：

```text
System Volume Information
$RECYCLE.BIN
Recovery
Config.Msi
```

跳过的文件不会进入内存中的扫描结果，也不会出现在界面表格和导出报告中。概览和报告会统计跳过文件数、跳过目录数和跳过原因。

## 超时保护

为避免扫描长时间停在单个文件上，当前支持“单文件超时”设置：

```text
默认：15 秒
0：关闭超时保护
```

如果读取文件元数据或计算 Hash 超过阈值，程序会：

- 在扫描错误页记录该文件。
- 在跳过原因里记录“慢文件”或“慢 Hash”。
- 继续扫描后续文件。

注意：如果操作系统底层调用本身长时间不返回，程序只能在这次调用返回后判断超时；普通 Python 代码不能强行中断已经卡死的系统调用。

## 只扫描规则

开启“只扫描匹配规则”后，只有命中以下任一规则的文件才会进入结果：

- 扩展名。
- 文件名关键词。
- 路径关键词。
- 文件类型，例如 `图片`、`文档`、`代码`、`可执行`。

跳过规则固定优先于只扫描规则。也就是说，文件即使命中只扫描规则，只要同时命中跳过规则，仍会被跳过。

## 报告导出

报告导出会生成一个带时间戳的目录，包含：

```text
summary.csv      # 扫描概览
files.csv        # 文件明细
duplicates.csv   # 重复文件
risks.csv        # 可疑文件
errors.csv       # 扫描错误
report.html      # 带图表的 HTML 报告
```

CSV 使用 `utf-8-sig` 编码，方便用 Excel 打开中文内容。设置页可以关闭“报告导出完整路径”，关闭后 CSV 和 HTML 报告只写文件名或目录名，降低完整路径泄露风险。

## 设置文件

设置保存到用户目录：

```text
%USERPROFILE%\.fileaudit\settings.json
```

如果设置文件损坏，程序会自动回退到默认设置。

## 时间图表

扫描概览和 HTML 报告里的“修改时间分布”默认按以下范围分类：

```text
今天
7 天内
30 天内
3 个月内
更早
未来时间
```

其中“3 个月内”可以在设置页的“修改时间分类”中调整。

## 已知问题与处理

### 大目录扫描内存较高

程序会保留完整扫描结果用于统计、筛选和报告导出。为降低 UI 内存压力，文件明细、重复文件、可疑文件和扫描错误表格最多先渲染前 5000 行，并通过“查看更多 1000 行”继续加载。

后续如果继续优化，应改成分页或 `QTableView + QAbstractTableModel` 虚拟表格。

### Windows 目录看似卡住

扫描 `C:\Windows` 时，可能在 `.mui` 等系统资源文件处停留较久。原因通常不是目录遍历卡死，而是进入了重复文件 Hash 阶段。

当前底部状态栏会区分“正在扫描文件”和“正在计算重复文件 Hash”，Hash 阶段会显示 `当前 / 总数` 和当前处理文件，Hash 读取过程中也会检查取消请求。

### 受保护目录拒绝访问

Windows 的 `System Volume Information`、`$RECYCLE.BIN`、`Recovery` 等目录通常需要系统权限。当前默认会跳过这些目录；如果仍遇到 `WinError 5`，扫描错误页会显示更友好的“无权限访问”提示。

### 表格排序崩溃

开发过程中尝试过自定义 `QTableWidgetItem.__lt__` 来做真实大小排序，但 PySide6 在原生排序时触发过进程级内存读取错误。

当前不再启用 Qt 内置排序。文件明细页点击表头时，由 Python 对数据列表排序后重新绘制表格。
